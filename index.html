Tiered Hybrid

  1. Isolate & Migrate loosely coupled front-end apps to cloud
  2. Re-use existing backends on-premise as can be more complex with more complex dependencies
  3. Front-ends rapidly change & cloud simplofy setup of CI/CD pipelines
  4. unidirectional communication

Analytics Hybrid

  1. Cost savings as analytic workloads can be bursty & process substantial data amounts
  2. Cloud Storage is better suited for building data lakes

DR Hybrid

  1. Use public cloud as a DR failover environment
  2. Not often used as hard to keep identical infrastructures on-prem and in public cloud
